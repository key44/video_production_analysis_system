# コントリビュートガイドライン / Contributing Guidelines

（🇬🇧 English guidelines are below the Japanese section）

開発にご関心をお寄せいただき、ありがとうございます！このプロジェクトは、日本の中小企業向けに「外部（クラウド・サーバー）へ情報を一切送信しない」ローカル完結型の動作分析システムを提供することを目的としています。

皆さまからの活用事例の共有、機能要望、バグリポート、そしてプルリクエスト（PR）を歓迎しています。

## 活用事例の共有について

このツールが現場でどのように使われているかを知ることは、開発の大きなモチベーションになります！
**GitHub Discussions** の「Show and tell」カテゴリにて、ぜひ皆さまの活用事例を教えてください。

- どのような工程・業種で使ったか
- どんな改善効果があったか
- こういう機能があったらもっと便利だと思ったこと

などを気軽に投稿してください。

## バグ報告の手順

バグを発見した場合は、**GitHub Issues** から報告をお願いします。
報告の際は「Bug report」テンプレートを使用し、以下の情報を含めてください：
- 再現手順
- 期待される動作と実際の動作
- 実行環境（OS、ブラウザなど）

## 機能要望について

「こんな機能が欲しい」という要望も大歓迎です！
**GitHub Issues** にて「Feature request」テンプレートを使用し、ラベル `enhancement` を付けて投稿してください。
その際、「どのような課題を解決したいか」「現場でどのように活用したいか」を具体的に書いていただけると、検討がスムーズになります。

## プルリクエスト（PR）のガイドライン

コード修正や機能追加のPRも受け付けています。以下の手順でお願いします：
1. このリポジトリをフォークする
2. 新しい機能ブランチを作成する (`git checkout -b feature/amazing-feature`)
3. 変更をコミットする (`git commit -m 'Add some amazing feature'`)
4. ブランチにプッシュする (`git push origin feature/amazing-feature`)
5. Pull Request を作成する

既存のコードには変更を加えないよう、機能追加のみのPRを優先して確認します。

本システムは **GPL-3.0** ライセンスのもとで公開されています。
あなたが提供された貢献物（コード、ドキュメントなど）も、本プロジェクトの一部として GPL-3.0 ライセンスの下で配布されることに同意したものとみなされます。あらかじめご了承ください。

## 🇬🇧 English: Contributing Guidelines

Thank you for your interest in contributing! This project provides a local-only video production analysis system designed primarily for Japanese SMEs, ensuring that no data is ever sent to external cloud servers.

We welcome use-case sharing, feature requests, bug reports, and Pull Requests.

### Sharing Your Use Case
We'd love to hear how you use this tool! Please share your stories in the "Show and tell" category on **GitHub Discussions**. 

### Bug Reports & Feature Requests
Please use **GitHub Issues** to report bugs or request new features. We have templates prepared to help you provide the necessary information. For feature requests, please make sure the `enhancement` label is applied.

### Pull Requests
1. Fork the repo and create your feature branch.
2. Commit your changes.
3. Push to the branch and open a Pull Request.
